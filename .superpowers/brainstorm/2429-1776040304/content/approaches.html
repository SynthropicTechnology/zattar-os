3 Abordagens para o Redesign do Comunica CNJ

Escolha a direção arquitetural que melhor serve o dia-a-dia do escritório

A

🏛️ "Gazette Command Center"

Inspiração: Stripe Dashboard + Linear Triage

Dashboard-first. KPIs interativos no topo (clicáveis = filtram a tabela). Abaixo, tabela de alta densidade com view tabs (Todas, Pendentes, Órfãos, Prazos). Detail panel deslizante à direita. Painel de resolução de órfãos como uma view dedicada com split-panel e scores de confiança.

Layout:
┌─────────────────────────────────────────────────┐
│  Header + NLP Search Bar                        │
├────────┬────────┬────────┬────────┬────────────┤
│  KPI 1 │  KPI 2 │  KPI 3 │  KPI 4 │   KPI 5   │
├────────┴────────┴────────┴────────┴────────────┤
│  [Todas] [Pendentes] [Órfãos] [Prazos] [+View] │
│  [Fonte ▼] [Tipo ▼] [Status ▼]  chips ativos   │
├─────────────────────────────────┬───────────────┤
│  Data Table (alta densidade)    │ Detail Panel  │
│  - Tipo, Processo, Parte,       │ (slide-in)    │
│    Fonte, Data, Prazo, Status   │ - Texto       │
│  - Inline quick-actions         │ - PDF preview │
│  - Row hover → expand preview   │ - Timeline    │
│                                 │ - Vincular    │
└─────────────────────────────────┴───────────────┘
✓ Máxima densidade de informação ✓ KPIs funcionais (filtram ao clicar) ✓ Keyboard-first para processamento rápido
⚠ Curva de aprendizado para usuários novos ⚠ Pode parecer "fria" sem personalidade visual
B

📬 "Gazette Inbox"

Inspiração: Intercom Inbox + Missive + QuickBooks Reconciliation

Inbox-first. Lista de publicações como feed à esquerda com preview cards (não tabela). Painel de detalhes à direita (sempre visível em desktop). KPIs como uma faixa compacta acima da lista. Workflow de triagem inline — badges de status editáveis direto na lista. Resolução de órfãos com confidence scores integrado no painel de detalhes.

Layout:
┌─────────────────────────────────────────────────┐
│  Header + NLP Search + Filter Chips             │
├────────┬────────┬────────┬────────┬────────────┤
│  KPI 1 │  KPI 2 │  KPI 3 │  KPI 4 │   KPI 5   │
├────────┴────────┴────────┴────────┴────────────┤
│  [Todas] [Pendentes] [Órfãos] [Prazos]         │
├─────────────────────┬───────────────────────────┤
│  Publication Cards   │  Detail Panel             │
│  ┌───────────────┐  │  ┌─────────────────────┐  │
│  │ ● INTIMAÇÃO   │  │  │  Texto completo      │  │
│  │ TJ-PR • 12/04 │  │  │  ─────────────────   │  │
│  │ Silva vs Sant. │  │  │  Processo vinculado  │  │
│  │ ○ Pendente    │  │  │  Timeline do caso    │  │
│  └───────────────┘  │  │  PDF inline           │  │
│  ┌───────────────┐  │  │  Match suggestions    │  │
│  │ ● DESPACHO    │  │  │  [Vincular] [Ignorar] │  │
│  │ ...           │  │  └─────────────────────┘  │

✓ UX intuitiva (parece email/inbox) ✓ Preview cards mais ricos que tabela ✓ Detail panel sempre visível = contexto máximo
⚠ Menos itens visíveis por vez (cards ocupam mais espaço) ⚠ Processamento em batch mais lento que tabela
C

⚡ "Gazette Fusion" (Recomendado)

Inspiração: Stripe + Linear + Notion + QuickBooks — o melhor de cada

Fusão inteligente. KPIs interativos no topo (Stripe). View tabs como presets salvos (Notion). Tabela de alta densidade como modo padrão MAS com toggle para modo cards (Linear). Detail panel deslizante (Linear). View "Órfãos" com split-panel de resolução e confidence scores (QuickBooks). NLP search progressivo. Tudo envolvido em Glass Briefing com profundidade semântica.

Layout:
┌─────────────────────────────────────────────────┐
│  🔍 NLP Search (progressivo: texto → filtros)   │
├────────┬────────┬────────┬────────┬────────────┤
│▸KPI 1  │ KPI 2  │ KPI 3  │ KPI 4  │  KPI 5    │
│ click→ │ click→ │ click→ │ click→ │  click→   │
│ filtra │ filtra │ filtra │ filtra │  filtra   │
├────────┴────────┴────────┴────────┴────────────┤
│ [Todas][Pendentes][Órfãos][Prazos][+View] ⊞⊟  │
│ Chips: [TJ-PR ×] [Intimação ×]     [Limpar]   │
├─────────────────────────────┬───────────────────┤
│ Table/Cards (toggle ⊞/⊟)   │ Detail Panel      │
│                             │ (contextual):     │
│ TABLE MODE: alta densidade  │                   │
│ com inline actions          │ Normal: texto +   │
│                             │ PDF + timeline    │
│ — OU —                      │                   │
│                             │ Órfão: split com  │
│ CARD MODE: preview rico     │ match scores +    │
│ com excerpt + badges        │ [Vincular/Ignorar]│
│                             │                   │
│ Keyboard: ↑↓ navega,       │ AI: resumo auto + │
│ Enter abre, / busca        │ prazos destacados  │
└─────────────────────────────┴───────────────────┘
✓ Flexível: tabela para volume, cards para review ✓ KPIs funcionais + views salvos ✓ Resolução de órfãos como view dedicada ✓ AI integrada (NLP + matching + resumos) ✓ Glass Briefing nativo
⚠ Mais complexo de implementar (maior escopo) ⚠ Precisa de design cuidadoso para não sobrecarregar

💡 Por que recomendo C (Fusion)?

  • Adaptável: advogados que processam volume usam tabela; quem faz triagem detalhada usa cards
  • Progressivo: começa simples (KPIs + tabela), revela complexidade sob demanda (filtros, AI, views)
  • Órfãos como feature: não é um "bug" a resolver, é uma view de trabalho com UX dedicada
  • Glass Briefing: cada camada (KPI strip → filter bar → tabela → detail panel) mapeia para um depth diferente do GlassPanel
  • Referências sólidas: cada pedaço vem de um padrão comprovado, não é invenção especulativa
